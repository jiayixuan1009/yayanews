### 1.2 🔴 P0：管理后台存在 RCE 隐患

[apps/admin/src/app/api/benchmarks/route.ts](apps/admin/src/app/api/benchmarks/route.ts#L23-L43)：

```ts
const cmd = `python -m pipeline.tools.speed_benchmark --limit 10 --hours 24`;
exec(cmd, { cwd: projectRoot, timeout: 120_000 }, ...)
```

虽然当前命令行是常量，但：
- 使用 `exec()`（启 shell）而非 `execFile()`/`spawn()`，未来一旦混入 query 参数即变 RCE。
- 异常分支会向客户端回传 `stdout`、`stderr`、`err.message`，相当于把服务器内部错误细节暴露给攻击者（依赖 `requireAuth`，但 `ADMIN_API_TOKEN` 一旦泄漏即沦陷）。

**修复**：改为 `spawn('python', ['-m', 'pipeline.tools.speed_benchmark', '--limit', '10', '--hours', '24'], { cwd: projectRoot, timeout: 120_000 })`，并将 stderr 仅写入服务端日志，不回传给前端。

from urllib.parse import urlparse
import ipaddress, socket

if ip.is_private or ip.is_loopback or ip.is_link_local:

## 2. 代码质量

### 2.1 🟠 高：缺 ESLint 配置

`apps/web`、`apps/admin` 均未发现 `.eslintrc.*` / `eslint.config.*`，`npm run lint` 实际只跑 `next lint` 内置规则。建议在根目录新增 `eslint.config.mjs`（flat config），开启：
- `@typescript-eslint/no-explicit-any`
- `no-console`（生产源代码区域）
- `import/order`
- `eqeqeq`、`no-unused-vars`

### 2.2 🟠 高：`as any` 与字典类型

子代理统计 37 处 `as any`，集中在：
- [apps/web/src/app/[lang]/page.tsx](apps/web/src/app/[lang]/page.tsx#L92)（多处 `(dict.nav as any)`、`(dict.home as any)`）
- [apps/web/src/lib/analytics.ts](apps/web/src/lib/analytics.ts#L17-L18) `(window as any).gtag`
- [packages/seo/src/metadata.ts](packages/seo/src/metadata.ts#L101)、[packages/seo/src/json-ld.ts](packages/seo/src/json-ld.ts#L20)
- [infra/scripts/google-indexer.ts](infra/scripts/google-indexer.ts#L102-L107)

**修复方向**：
- 给 `dictionaries` 输出补 `Dictionary` 类型（在 `@yayanews/types` 中沉淀）
- `window.gtag` 用 `declare global { interface Window { gtag?: (...args: unknown[]) => void } }`
- tsconfig 增加 `"noUncheckedIndexedAccess": true` 后，再批量替换为安全访问

| 位置 | 现状 |
|------|------|
